Branch office network lab with VLAN segmentation, router-on-a-stick, and ACL enforcement.

This GitHub Pages project documents a small office network built to demonstrate practical CCNA-level skills: VLAN creation, 802.1Q trunking, inter-VLAN routing through router subinterfaces, and controlled guest isolation from internal server resources.

Topology

Small branch office layout

R1 connects to SW1 over a trunk, and SW1 extends VLANs to SW2 over a second trunk. User devices sit in VLAN 10, the internal server sits in VLAN 20, and the guest endpoint sits in VLAN 30.

Core devices

  • R1 router with subinterfaces on G0/0
  • SW1 as the primary access and trunk switch
  • SW2 as the downstream access switch

Endpoint placement

  • PC1 and PC2 on SW1 in VLAN 10
  • PC3 and PC4 on SW2 in VLAN 10
  • Server1 on SW1 in VLAN 20; Guest-PC on SW2 in VLAN 30
Addressing

VLAN and IP plan

The lab uses simple /24 networks so the routing logic stays easy to validate and explain in a portfolio review.

VLAN Name Subnet Gateway Purpose
10 USERS 192.168.10.0/24 192.168.10.1 Employee endpoints
20 SERVERS 192.168.20.0/24 192.168.20.1 Internal server segment
30 GUEST 192.168.30.0/24 192.168.30.1 Guest access
99 MGMT 192.168.99.0/24 192.168.99.1 Optional management
Ports

Switch port assignments

SW1

  • Fa0/1 → PC1 → VLAN 10
  • Fa0/2 → PC2 → VLAN 10
  • Fa0/10 → Server1 → VLAN 20
  • Fa0/23 → trunk to SW2
  • Fa0/24 → trunk to R1

SW2

  • Fa0/1 → PC3 → VLAN 10
  • Fa0/2 → PC4 → VLAN 10
  • Fa0/10 → Guest-PC → VLAN 30
  • Fa0/24 → trunk to SW1
Configurations

Core device configuration examples

These snippets are sized for a portfolio page. In the repository, keep full device configs in a separate configs/ folder.

SW1

enable
configure terminal
hostname SW1

vlan 10
 name USERS
vlan 20
 name SERVERS
vlan 30
 name GUEST
vlan 99
 name MGMT

interface range fa0/1 - 2
 switchport mode access
 switchport access vlan 10
 spanning-tree portfast
 no shutdown

interface fa0/10
 switchport mode access
 switchport access vlan 20
 spanning-tree portfast
 no shutdown

interface fa0/23
 switchport mode trunk
 switchport trunk allowed vlan 10,20,30,99
 no shutdown

interface fa0/24
 switchport mode trunk
 switchport trunk allowed vlan 10,20,30,99
 no shutdown

R1

enable
configure terminal
hostname R1

interface g0/0
 no ip address
 no shutdown

interface g0/0.10
 encapsulation dot1Q 10
 ip address 192.168.10.1 255.255.255.0

interface g0/0.20
 encapsulation dot1Q 20
 ip address 192.168.20.1 255.255.255.0

interface g0/0.30
 encapsulation dot1Q 30
 ip address 192.168.30.1 255.255.255.0

ip access-list extended GUEST_BLOCK_SERVERS
 deny ip 192.168.30.0 0.0.0.255 192.168.20.0 0.0.0.255
 permit ip any any

interface g0/0.30
 ip access-group GUEST_BLOCK_SERVERS in
Validation

Verification and expected results

Show commands

show vlan brief
show interfaces trunk
show interfaces status
show ip interface brief
show ip route
show access-lists

These commands confirm VLAN membership, trunk state, subinterface status, routing table entries, and ACL logic.

Ping tests

  • PC1 to Server1: should succeed
  • PC3 to PC1: should succeed
  • Guest-PC to 192.168.30.1: should succeed
  • Guest-PC to Server1: should fail

Showing both permitted and blocked traffic makes the project much stronger for interviews and resume discussions.
